GitHub Copilot app is now available in technical preview

GitHub Changelog Thu, 14 May 2026 16:10:33 +0000

GitHub has released a technical preview of the GitHub Copilot app, marking a significant step toward fully agentic development on the desktop. The app is described as a GitHub-native experience, meaning it integrates directly with repositories, issues, and pull requests rather than operating as a standalone editor plugin. Its core design philosophy is to let developers start agentic coding sessions from the work already in front of them, such as open issues or pending PRs. The app is built to keep agentic sessions isolated, preventing runaway changes from affecting unrelated parts of a codebase. Developers can steer the agent mid-session, giving them control over how autonomous tasks unfold. This positions the Copilot app as a distinct product from the Copilot extension inside VS Code, targeting a workflow that begins with project management context. The technical preview phase means early adopters can test the experience and provide feedback before broader availability. This release reflects GitHub's broader push to make Copilot a first-class agentic platform, not just an autocomplete assistant. The app complements other recent agentic Copilot investments, including cloud agent auto model selection and JetBrains integration updates. For professional developers, this could fundamentally change how they delegate and supervise coding tasks. It also signals GitHub's intention to compete more directly with standalone AI coding environments like Cursor and Windsurf.

Impact Takeaways

  • A new GitHub-native desktop app brings agentic Copilot sessions outside of VS Code for the first time, expanding where developers can delegate autonomous coding tasks.
  • The isolated session design reduces risk when running agentic workflows on complex codebases, addressing a major concern around AI autonomy.
  • Teams in the technical preview can shape the product roadmap early, making it a high-value opportunity for developer experience leads and platform engineers.

Copilot Memory supports user preferences for Pro, Pro+ users

GitHub Changelog Fri, 15 May 2026 17:14:17 +0000

GitHub Changelog announced that Copilot Memory now supports user-level preferences, currently available in early access for Copilot Pro and Pro+ users. This feature enables Copilot to store personal preferences about how a user likes to work, including both preferences that are explicitly told to Copilot and those that are inferred from patterns of interaction. The capability represents a meaningful step toward a truly personalized AI coding assistant that adapts to individual developer habits over time. Previously, Copilot sessions were largely stateless from a personalization standpoint, requiring users to repeatedly specify preferences in system prompts or custom instructions. With persistent memory, preferences such as preferred coding style, language settings, testing frameworks, or communication tone can be retained across sessions. The early access rollout to Pro and Pro+ tiers suggests GitHub is gathering feedback before a wider release, likely including enterprise tiers. Privacy considerations are inherent to any persistent memory feature, and GitHub will need to communicate clearly how stored preferences can be reviewed, edited, or deleted. This feature positions Copilot more competitively against tools like Cursor and Claude, which have offered various forms of persistent context and memory. For individual developers, the practical benefit is reduced friction in every new session, as Copilot will already know their preferences without re-prompting. Teams with standardized workflows could also use stated preferences to align Copilot's behavior with team conventions. This is among the most impactful UX improvements to individual Copilot plans in recent months.

Impact Takeaways

  • Pro and Pro+ users in early access can now benefit from a Copilot that remembers coding preferences across sessions, reducing repetitive setup in every conversation.
  • Inferred preference storage means Copilot can adapt even when users don't explicitly configure settings, lowering the barrier to personalization.
  • Developers should familiarize themselves with how to view and manage stored memory to maintain control over what Copilot retains about their preferences.

Introducing Copilot CLI agent and unified sessions view in GitHub Copilot for JetBrains IDEs

GitHub Changelog Thu, 14 May 2026 02:38:40 +0000

GitHub has released a significant update to the GitHub Copilot plugin for JetBrains IDEs, introducing the Copilot CLI agent capability directly within the IDE environment. This allows JetBrains users—including those on IntelliJ IDEA, PyCharm, WebStorm, and other JetBrains products—to run agentic coding tasks without leaving their editor. A new unified sessions view accompanies the CLI agent, giving developers a consolidated interface to monitor the live status of running and queued Copilot agent sessions. The ask mode, previously more prominent in VS Code's Copilot Chat experience, is also now available in JetBrains, enabling conversational coding assistance without requiring edits. These additions represent a meaningful step toward feature parity between the VS Code and JetBrains Copilot experiences, which has historically lagged. For organizations where JetBrains IDEs are the standard—particularly in Java, Kotlin, Python, and web development shops—this update unlocks agentic workflows that were previously only practical in VS Code. The unified sessions view is especially valuable for managing long-running or parallel agent tasks, reducing the cognitive overhead of tracking what the AI is doing. The Copilot CLI agent integration also hints at tighter coordination between terminal-based automation and IDE-based editing in future releases. This update aligns with the broader GitHub strategy of making Copilot's agentic capabilities available across all major development environments. JetBrains users should update their Copilot plugin to access these new features immediately.

Impact Takeaways

  • JetBrains developers can now run Copilot CLI agent sessions directly inside their IDE, eliminating the need to switch to VS Code for agentic coding workflows.
  • The unified sessions view gives teams better visibility into active and queued agent tasks, which is essential for managing multi-step autonomous coding operations.
  • The addition of ask mode to JetBrains closes a long-standing feature gap, making Copilot's conversational assistance fully usable without triggering edit operations.
