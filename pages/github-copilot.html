GitHub Copilot is moving to usage-based billing

GitHub Blog Mon, 27 Apr 2026 15:58:22 +0000

GitHub has announced that beginning June 1, 2026, Copilot usage will be measured and billed through GitHub AI Credits, marking a fundamental shift away from flat-rate subscription pricing. This usage-based model is designed to align costs more closely with actual consumption, potentially benefiting light users while increasing costs for heavy users. The move is part of a broader GitHub platform strategy to unify AI resource consumption under a single credit system. This change affects all Copilot tiers and integrates with the concurrent announcement that Copilot code review will begin consuming GitHub Actions minutes on the same date. Organizations and individual developers will need to monitor their usage patterns to forecast and manage expenses effectively. The GitHub AI Credits system is expected to cover not just Copilot chat and code completions, but also agentic features and code review capabilities. GitHub noted that the transition is intended to provide more flexibility and transparency in how AI features are consumed. Teams with variable or seasonal development activity may find the new model more economical than fixed plans. The announcement was accompanied by related changes to Copilot Individual plans, signaling a comprehensive restructuring of Copilot's commercial offerings. Developers are advised to review their usage dashboards ahead of the June 1 cutover to avoid billing surprises.

Impact Takeaways

  • Starting June 1, 2026, Copilot costs will be metered via GitHub AI Credits — teams must audit usage now to forecast new spend.
  • Usage-based billing may reduce costs for infrequent users but could significantly increase expenses for power users and large engineering teams.
  • The change is bundled with Copilot code review also consuming GitHub Actions minutes, compounding the billing impact for teams using agentic review features.

GitHub Copilot code review will start consuming GitHub Actions minutes on June 1, 2026

GitHub Changelog Mon, 27 Apr 2026 15:59:24 +0000

GitHub has announced that its Copilot code review feature — which provides AI-powered, agent-driven reviews on pull requests — will begin drawing from GitHub Actions minutes starting June 1, 2026. This change means that teams currently using Copilot code review at no additional compute cost will need to account for Actions minutes consumption going forward. GitHub explicitly acknowledged the significance of this change for customers, particularly engineering teams with high pull request volumes. The announcement is closely tied to the broader shift to usage-based billing for Copilot, with both changes taking effect on the same date. Copilot code review has grown popular as a way to get consistent, automated feedback on every PR without requiring senior developer time for initial review passes. Teams will now need to factor Actions minutes into their cost modeling for this feature. GitHub has not yet published a detailed breakdown of how many minutes a typical code review consumes, which may complicate budget forecasting. Organizations should audit their monthly PR volumes and current Actions minutes budgets ahead of the transition. The change reflects GitHub's move to treat all agentic, compute-intensive Copilot features as metered resources rather than flat-rate inclusions. Customers are advised to review GitHub's documentation and pricing pages for updated guidance before the June 1 deadline.

Impact Takeaways

  • Copilot code review will consume GitHub Actions minutes from June 1 — teams with high PR volume should audit their Actions budget immediately.
  • This change, combined with usage-based Copilot billing, could substantially increase AI tooling costs for organizations relying on automated code review at scale.
  • GitHub has not yet clarified per-review Actions minute consumption rates, making cost forecasting difficult — watch for updated documentation before the cutover date.
